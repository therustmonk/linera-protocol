source ·
Expand description

A storage backend for views based on ScyllaDB This provides a KeyValueStoreClient for the ScyllaDB database. The code is functional but some aspects are missing.

The current connection is done via a Session and a corresponding primary key that we name table_name. The maximum number of concurrent queries is controlled by max_concurrent_queries.

We thus implement the

  • KeyValueStoreClient for a database access.
  • Context for the context.

Support for ScyllaDB is experimental and is still missing important features: TODO(#935): Read several keys at once TODO(#934): Journaling operations
